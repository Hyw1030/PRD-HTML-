# PRD HTML 本地整理工具

一个纯前端、本地运行的 PRD / Axure HTML 整理工具。

## 能做什么

- 选择单个 HTML 文件，或选择 Axure 导出的完整 HTML 文件夹。
- 支持选择一个总目录，批量识别多个需求包。
- 在浏览器本地抽取 HTML 文本、Axure 控件文字、原型标注和图片数量。
- 生成 Markdown 草稿，便于后续人工整理或导入知识库。
- 不上传文件，不调用 API。

## 不能做什么

这是不调用 AI 的本地规则版，所以它不能稳定完成真正的产品理解总结：

- 不能真正理解截图里的页面内容。
- 不能像产品经理一样稳定归纳“需求做了什么、原型改了什么”。
- 不能保证生成内容可直接作为最终知识库文档。

如果要得到高质量总结版 Markdown，建议在此工具的基础上增加 LLM / OCR / 视觉模型。

## 使用方式

直接双击打开：

```text
index.html
```

然后点击：

```text
选择 PRD / Axure 文件夹
```

选择一个 HTML 文件夹或总目录即可。

## 输出

工具会在页面中展示每个需求包的 Markdown 草稿，并支持：

- 下载当前 Markdown
- 复制当前 Markdown
- 下载全部 Markdown

## 开源说明

本项目为纯前端工具，无后端服务，无 API Key。

如果后续接入 AI 总结能力，请不要把个人 API Key 写入代码。建议使用服务端代理或让使用者自行配置 Key。
